# KPI 売上 CSV — ダミーデータ作成指示書（AI 用）

**根拠コード**: `app/Services/CsvImportService.php`（`parseCsvToSalesRecords` / `inferCsvStatus`）、`app/Http/Requests/Admin/CsvUploadRequest.php`（アップロード制約）

この文書を AI にそのまま渡し、**UTF-8 の `.csv` ファイル**を生成させる。目的は **取込成功件数・失敗件数・OK 率表示**の確認用ダミーデータ。

---

## 1. アップロード制約（HTTP）

| 項目 | 値 |
|------|-----|
| 拡張子 | `.csv` または `.txt`（`mimes:csv,txt`） |
| 最大サイズ | **10,240 KB（10MB）** |
| エンコーディング | **UTF-8** 推奨（日本語ヘッダ・本文を含むため） |

---

## 2. CSV 構造（必須）

1. **1 行目はヘッダ**（列名）。`str_getcsv` でパースされる標準 CSV（カンマ区切り、ダブルクォートでエスケープ可）。
2. **2 行目以降がデータ**。空行は無視される。
3. ヘッダ名は **次表の「いずれか 1 つ」をその文字列どおり** 使う（エイリアスは **完全一致** でマッチ。大文字小文字は区別される）。

### 2.1 列と認識されるヘッダ名（いずれか 1 列に相当）

| 論理名 | 認識されるヘッダ名（候補のどれか 1 つ） | 必須 | 備考 |
|--------|------------------------------------------|------|------|
| 日付 | `date` **または** `日付` | **必須** | 形式 **`YYYY-MM-DD`** のみ有効。それ以外は行エラー「日付が不正です」。 |
| 担当者 | `staff_name` **または** `担当者` **または** `スタッフ` | **必須** | 空は行エラー「担当者が空です」。 |
| 店舗名 | `store_name` **または** `店舗名` **または** `店名` | 任意 | 空文字可。 |
| 売上額 | `sales_amount` **または** `売上額` **または** `売上` | 任意 | **数字以外は除去**され整数化。負は `max(0, …)` で 0。 |
| 客数 | `customer_count` **または** `客数` | 任意 | 同上（非数字除去、`max(0, …)`）。 |
| 商材 | `product_name` **または** `商材` **または** `商品` | 任意 | 空は DB では `null`。 |
| 契約種別 | `contract_type` **または** `契約種別` | 任意 | 空は `null`。 |
| 流入 | `channel` **または** `流入` **または** `チャネル` | 任意 | 空は `null`。 |
| ステータス | `status` **または** `ステータス` **または** `判定` **または** `合否` | 任意 | 下記「3. OK / NG の決め方」参照。 |
| 結果 | `result` **または** `結果` **または** `成約` | 任意 | `status` が空のとき **ここから推測** される（下記）。 |

**注意**: ヘッダは **重複キー**にしない（同じ名前が 2 列あると後勝ちでマップが壊れる）。

---

## 3. OK / NG（`status` カラム）の決め方

実装は `inferCsvStatus($status列, $result列)`。

### 3.1 `status`（または `ステータス` / `判定` / `合否`）列が埋まっている場合

`mb_strtolower(trim(...))` 後の値が次に一致すればそのまま採用:

| 入力（例） | 保存値 |
|------------|--------|
| `ng`, `×`, `x`, `false`, `0` | `ng` |
| `ok`, `○`, `o`, `true`, `1` | `ok` |

### 3.2 上記に当てはまらない、または `status` 系が空の場合

`result`（または `結果` / `成約`）列の文字列に部分一致（`mb_strtolower` 後）:

**NG 扱い**（先にマッチした方が優先されるわけではなく、NG 側ループでヒットしたら `ng`）:

- 部分文字列: `ng`, `否`, `不成立`, `失敗`, `却下`, `未成約`

**OK 扱い**:

- 部分文字列: `成約`, `ok`, `成立`, `成功`

**どちらにも当てはまらない、または `result` も空**: 既定 **`ok`**

---

## 4. 取込結果の数え方（UI の「成功 / 失敗」）

- **成功**: 上記バリデーションを通過した行数 → `success_count`
- **失敗**: 日付不正 or 担当者空 の行数 → `failed_count`
- **総行数（進捗 UI）**: `success_count + failed_count`（= データ行数。ヘッダは含まない）

OK 率を見たい場合は、**成功行に有効な日付・非空担当者**を入れ、失敗は意図的に日付や担当者を欠けさせる。

---

## 5. 推奨ヘッダ行（日本語・1 行コピペ用）

```csv
日付,担当者,店舗名,売上,客数,商材,契約種別,流入,合否,成約
```

※ 売上額の候補は `売上額` または `売上`。**`売上`** なら上記で可。

---

## 6. 最小サンプル（全日成功）

```csv
日付,担当者,店舗名,売上,客数,商材,契約種別,流入,合否,成約
2026-05-01,山田太郎,渋谷店,"1,234,567",42,光回線,新規,Web,ok,成約
2026-05-02,佐藤花子,新宿店,890000,30,携帯,MNP,店頭,OK,成立
```

---

## 7. 成功・失敗混在サンプル（OK 率確認用）

- **2 行目**（データ 1 行目）: 日付 `2026/05/01` → **失敗**（`YYYY-MM-DD` 以外）
- **3 行目**: 担当者空 → **失敗**
- **4・5 行目**: 正常

```csv
日付,担当者,店舗名,売上,客数,商材,契約種別,流入,合否,成約
2026/05/01,山田太郎,渋谷店,1000000,10,光回線,新規,Web,ok,成約
2026-05-03,,池袋店,0,0,—,—,—,,
2026-05-01,佐藤花子,新宿店,500000,5,携帯,MNP,店頭,ok,成約
2026-05-04,鈴木一郎,横浜店,2000000,20,電気,切替,電話,1,成功
```

期待: **成功 2、失敗 2**。エラーログの「行」は **ファイルの行番号**（1 行目＝ヘッダ、失敗は行 2 と行 3）。

## 8. AI へのコピペ用プロンプト（例）

以下を AI に貼る:

```
あなたは CSV を生成する。要件は次のとおり厳守すること。

1. 出力は UTF-8 の CSV テキストのみ（説明文不要）。先頭行はヘッダ。
2. ヘッダは次のいずれかに合わせる（日本語推奨）:
   日付,担当者,店舗名,売上,客数,商材,契約種別,流入,合否,成約
3. 日付列は必ず YYYY-MM-DD。担当者列は成功行では空にしない。
4. データ行をちょうど N 行生成する。うち S 行は成功、F 行は失敗（S+F=N）とし、失敗行は
   - 日付を YYYY-MM-DD 以外にする、または
   - 担当者を空にする
   のどちらかで作る。
5. 金額・客数にカンマや円記号を混ぜてよい（数字以外は除去される）。
6. 合否・成約は実装仕様に従い、ok/ng が判別できる値にする（docs/CSV-SALES-DUMMY-DATA-INSTRUCTIONS.md の「3. OK / NG」参照）。

N=__, S=__, F=__ に置き換えて実行。
```

`N`, `S`, `F` を数値で指定する。

---

## 9. 実装と指示書のズレに注意する点

- **CORS / CSP / Vite** はブラウザ環境の話であり、**CSV 列定義とは無関係**。
- 本ドキュメントは **`CsvImportService` の現行実装**に同期している。列追加・バリデーション変更時は本ファイルとコードの両方を更新すること。
