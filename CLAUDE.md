# LOM Sales Hub — Claude Code 憲法

## 基本ルール

- **回答・コメント・コミットメッセージはすべて日本語**で書くこと
- 不明点・仕様の判断が必要な場合は**必ず一旦確認**してから進めること。勝手に判断して実装を進めない
- 破壊的操作（rm, git push --force, DROP TABLE 等）は**必ず事前確認**すること
- コードコメントは WHY が明らかでない場合のみ、最小限で記述すること

## コード品質とディレクトリ構造

- **責務分離**: ロジックは Controller または Action クラスに書き、React コンポーネントは表示に専念させること（Fat Component 禁止）
- **ディレクトリ整合性**: `app/Models`, `resources/js/Pages` など既存の構造を無視した場所への新規ファイル作成を禁止する

## インクリメンタルな開発

- **一気実装の禁止**: 大きな機能は必ず「概要設計（設計図）」を先に提示し、合意を得てから 1 関数 / 1 コンポーネントずつ実装・確認すること

## Git 操作ルール

- **コミット粒度**: 1 機能につき 1 コミット
- **メッセージ形式**: `feat:`, `fix:`, `refactor:`, `chore:` などのプレフィックスを付け、内容を日本語で記述すること
- **履歴書き換え時の確認**: `git reset --hard` 等を提案する際は、必ず「stash するか一時コミットするか」をユーザーに確認してから実行すること

## 個人最適化ルール（Takuma Edition）

- **環境配慮**: 開発機はメモリ 8GB の MacBook Air。動作が重くなる提案（並列ビルド・大量プロセス起動等）は避け、軽量な開発体験を優先すること
- **エラー対応**: 画面が真っ白・500 エラーになった際は推測で直さず、まず `storage/logs/laravel.log` の最新エラーを確認することを最初のステップとすること
- **説明スタイル**: 専門用語だけでなく「扉が閉まっている」などの直感的な比喩を交えて状況を説明し、安心感を与えること

## プロジェクト概要

**LOM Sales Hub（LOM.Hub）**
営業オペの周知・案件 KPI・業務依頼・昼休憩・商材情報・マイページ・管理者機能を 1 ポータルに集約する社内向け Web アプリ。

**スタック**:
- PHP 8.3 / Laravel 13
- React 18 + TypeScript 5 + Vite 8
- Inertia.js v2（ハイブリッド UI）
- Tailwind CSS 3
- MySQL（開発: MAMP）
- Pest 4（テスト）

**開発起動**: `composer dev`（concurrently で serve + queue + pail + vite）

## 開発環境

- ローカル: MAMP（`http://localhost:8888`）
- DB: MySQL（MAMP 内蔵）
- 本番: Render（Docker デプロイ）

## 作業フロー

1. 実装前に仕様確認（不明点は質問）
2. 影響範囲を確認してから変更
3. Pest テストが壊れていないか確認
4. 完了時に変更点を日本語で簡潔に報告

## 禁止事項

- `git push --force`（確認なしで実行禁止）
- `rm -rf` 系コマンド（確認なしで実行禁止）
- `.env` ファイルへの機密情報の直書き
- 未確認の仕様を勝手に実装すること
